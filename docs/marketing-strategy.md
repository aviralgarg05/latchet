# Marketing Strategy

## Positioning

Latchet is a **task ledger for AI coding agents**, not a memory product and not a transcript exporter.

Core promise:

> decisions, failed attempts, env quirks, and next action in one place

## Audience

- Solo developers using multiple coding agents
- Indie hackers and consultants
- AI-native engineers working across long sessions and multiple branches

## Launch sequence

1. Open-source the spec, CLI, MCP server, fixtures, and examples.
2. Publish before/after demos around restart speed and repeated-failure reduction.
3. Seed integration templates for Codex, Claude Code, Gemini CLI, and Cursor.
4. Launch on GitHub, Hacker News, X, and relevant Reddit communities.

## Content pillars

- Decision log, not summary
- What failed should survive the session
- Env quirks are the real hidden memory
- Transcript portability is overrated
- Current task state matters more than raw conversation history

## Commercial expansion

- Keep the local ledger and basic adapters open source.
- Add optional sync, shared review, governance, and analytics later.
- Sell first to consultants and small teams, not enterprise buyers.
