# v0.1.0 — Task ledger for AI coding agents

**Latchet keeps decisions, failed attempts, env quirks, and the next action in one place for AI coding agents.**

## What this solves

Coding agents forget durable task state across sessions. Latchet stores what was decided, what failed, what env quirk mattered, and what's next — in repo-local `.taskledger/` files.

## What ships

- Append-only event ledger with derived `state.json` and `state.md`
- CLI: init, task create, log, import-session, import-command, show, next, verify, export, import, redact, diff
- MCP server with 9 tools (including `import_command` and `import_session`)
- Session importers: Codex, Claude Code, Cursor, Gemini CLI
- Command output importer for test/build failures
- Prompt exporters for all four agents + generic markdown
- Landing site: https://latchet.vercel.app
- 18 automated tests

## Install

```bash
git clone https://github.com/aviralgarg05/latchet.git
cd latchet
npm install
npm run build
```

## Quickstart

```bash
node packages/cli/dist/index.js init --name "My Project"
node packages/cli/dist/index.js task create auth --title "Fix auth tests" --goal "Stop repeated failures"
node packages/cli/dist/index.js log failure --task auth --summary "Fixture users missing organization_id"
node packages/cli/dist/index.js log next_action --task auth --summary "Patch fixtures and rerun tests" --priority high
node packages/cli/dist/index.js show --task auth
```

## Known limitations

- No npm package yet — run from source
- No live vendor hooks or auto-capture
- Session import is conservative (structured notes, not full transcripts)
- MCP requires manual stdio setup
- No cloud sync or web dashboard
