--- SHOW HN FIRST COMMENT ---

Hi HN — I built Latchet because I kept hitting the same failure across Codex/Claude Code sessions.

The problem isn't missing chat history. It's missing durable task state: what was decided, what failed, what env quirk mattered, and what's next.

Latchet stores that as an append-only event ledger in .taskledger/ inside your repo. Handoff to another agent is a derived export, not the primary artifact.

Demo: [YOUR_DEMO_LINK]
GitHub: https://github.com/aviralgarg05/latchet

Quickstart:
  npm install && npm run build
  node packages/cli/dist/index.js init
  node packages/cli/dist/index.js task create auth --title "Fix auth" --goal "Stop repeated failures"
  node packages/cli/dist/index.js log failure --task auth --summary "Fixture users missing org_id"
  node packages/cli/dist/index.js show --task auth

v0.1 is local-first, manual logging, no cloud. Happy to answer questions about scope and what's deliberately not built yet.
